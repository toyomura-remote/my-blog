ginを利用すると返されるデータがオブジェクト型でdata:{hoge:hoge}のように返される
フロントで、response.dataで取り出そうとするとオブジェクトが取り出される。
mapでほしい型は配列なのでそこでエラー

下記が解決方法

1. **Go側 (Gin)**: 拡張性のために `gin.H{"data": posts}` と包んで送る。
    
2. **Axios層**: `response.data` で、まずはHTTPボディ全体（JSONオブジェクト）を受け取る。
    
3. **インターフェース層**: `PostResponse` という「型」を定義して、`{ data: Post[] }` という形であることを教える。
    
4. **サービス層**: `response.data.data` と指定して、本当に欲しかった **「中身の配列」** だけを抽出して Component に届ける。

export defaultで渡すものは、読み込みの際に任意の名前を設定できる

**useEffectの特性**
useEffect の第一引数に渡す関数は、「何も返さない（void）」か「クリーンアップ用の関数を返す」 というルールがあります。

もし useEffect(async () => { ... }) と書いてしまうと、その関数は自動的に Promise（約束手形のようなオブジェクト）を返してしまいます。 Reactは「関数が返ってくるはずなのに、Promiseが返ってきたぞ！」と混乱してエラーを投げてしまうのです。
